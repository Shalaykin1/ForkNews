## Reporting a Vulnerability

Если вы обнаружили уязвимость в безопасности, пожалуйста, сообщите об этом ответственно.

### Как сообщить

**НЕ создавайте публичный Issue для уязвимостей безопасности.**

Вместо этого:

1. Отправьте email на: [ваш-email] (или создайте private security advisory)
2. Используйте GitHub Security Advisories (рекомендуется):
   - Перейдите на вкладку Security
   - Нажмите "Report a vulnerability"
   - Заполните форму с деталями

### Что включить в отчет

- Описание уязвимости
- Шаги для воспроизведения
- Потенциальное влияние
- Предлагаемое исправление (если есть)
- Ваши контактные данные для дальнейшего общения

### Что ожидать

- **Подтверждение** в течение 48 часов
- **Оценка** уязвимости в течение 7 дней
- **Исправление** в зависимости от серьезности:
  - Критические: 1-7 дней
  - Высокие: 7-14 дней
  - Средние: 14-30 дней
  - Низкие: В следующем релизе

### После исправления

1. Мы опубликуем security advisory
2. Выпустим патч с исправлением
3. Упомянем вас в благодарностях (если вы согласны)

## Security Best Practices для пользователей

### Установка

- Скачивайте APK только из официальных источников:
  - GitHub Releases этого репозитория
  - Google Play Store (когда будет доступно)

### Разрешения

Приложение запрашивает только необходимые разрешения:
- **INTERNET** - для проверки обновлений
- **POST_NOTIFICATIONS** - для отправки уведомлений
- **SCHEDULE_EXACT_ALARM** - для точного планирования

### Данные

- Все данные хранятся локально на устройстве
- Приложение не отправляет данные на сторонние серверы
- Запросы идут только к GitHub API и GameHub

### GitHub Token

Если добавляете GitHub токен:
- Используйте токен с минимальными правами (только `public_repo`)
- Не делитесь токеном с другими
- Регулярно обновляйте токен
- В текущей версии токен хранится в коде (будет исправлено)

## Security Features

### Текущие меры безопасности

1. **HTTPS Only**: Все сетевые запросы через HTTPS
2. **SQL Injection Protection**: Room автоматически защищает от SQL injection
3. **Input Validation**: Валидация URL перед парсингом
4. **ProGuard**: Обфускация кода в release версии
5. **No Hardcoded Secrets**: Нет хардкоженных токенов или ключей

### Planned Security Improvements

- [ ] Зашифрованное хранение токенов (EncryptedSharedPreferences)

## Известные ограничения

### GitHub API Rate Limiting

- Без токена: 60 запросов/час
- Возможна блокировка при превышении лимита
- Рекомендуется добавить токен для большого количества репозиториев

### GameHub Parsing

- Зависит от структуры HTML сайта
- Может сломаться при изменениях на сайте
- Нет официального API

### Third-party Dependencies

Приложение использует следующие библиотеки:
- Retrofit & OkHttp - сетевые запросы
- Room - база данных
- WorkManager - фоновые задачи
- Jsoup - HTML парсинг

Регулярно обновляем зависимости для исправления уязвимостей.
