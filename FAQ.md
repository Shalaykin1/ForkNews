# FAQ - Часто задаваемые вопросы

## Общие вопросы

### Что такое ForkNews?

ForkNews - это Android приложение для мониторинга обновлений в GitHub репозиториях и на сайте GameHub. Приложение автоматически проверяет наличие новых релизов и отправляет уведомления.

### Для какой версии Android подходит приложение?

Минимальная версия - Android 13 (API 33). Приложение будет работать на Android 13 и выше.

### Приложение бесплатное?

Да, ForkNews полностью бесплатное и с открытым исходным кодом (MIT License).

### Есть ли реклама в приложении?

Нет, в приложении нет никакой рекламы.

## Использование

### Как добавить GitHub репозиторий?

1. Нажмите на кнопку "+" в правом нижнем углу
2. Выберите "GitHub репозиторий"
3. Введите URL репозитория в формате: `https://github.com/owner/repo`
4. Нажмите "Добавить"

### Как добавить мониторинг GameHub?

1. Нажмите на кнопку "+"
2. Выберите "GameHub"
3. Репозиторий будет добавлен автоматически

### Как удалить репозиторий?

Свайпните карточку репозитория влево или вправо и подтвердите удаление.

### Как изменить частоту проверки?

1. Откройте меню → Настройки
2. Выберите нужную частоту в "Частота проверки"
3. Доступные варианты: 30 мин, 1 час, 2 часа, 6 часов, 12 часов

### Как настроить проверку в определенное время?

1. Откройте Настройки
2. Выберите "Пользовательское время" в "Частота проверки"
3. Нажмите "Выбрать время"
4. Установите нужное время (например, 9:00)

### Как отключить уведомления?

**Глобально:**
1. Настройки → Уведомления → Выключить

**Для конкретного репозитория:**
- Переключите переключатель "Уведомления" на карточке репозитория

### Как просмотреть релиз?

Нажмите на карточку репозитория - откроется браузер со страницей релиза.

### Что означает зеленый индикатор на карточке?

Зеленый индикатор означает, что вышел новый релиз. Он исчезнет после того, как вы откроете страницу релиза.

## Темы

### Как изменить тему?

1. Настройки → Тема
2. Выключите "Использовать системную тему"
3. Включите "Темная тема" для темной темы

### Как включить автоматическое переключение темы?

Настройки → "Использовать системную тему" → Включить

Тема будет автоматически меняться в соответствии с системными настройками.

## Технические вопросы

### Сколько репозиториев можно добавить?

Технически - неограниченное количество. Но помните о лимитах GitHub API (см. ниже).

### Какие лимиты у GitHub API?

- **Без авторизации**: 60 запросов в час
- **С токеном**: 5000 запросов в час

Для большого количества репозиториев рекомендуется добавить GitHub токен.

### Как добавить GitHub токен?

В текущей версии это требует изменения кода. В будущих версиях планируется добавить в настройки.

### Почему GameHub не работает?

GameHub использует парсинг HTML страницы. Если структура сайта изменилась, парсинг может сломаться. Сообщите об этом в Issues.

### Приложение потребляет много батареи?

Нет. Приложение использует WorkManager, который оптимизирует фоновые проверки и объединяет их с другими задачами системы.

### Где хранятся данные?

Все данные хранятся локально на устройстве в базе данных Room и DataStore Preferences.

### Передаются ли данные на сервер?

Нет, приложение не передает никаких данных на внешние серверы (кроме запросов к GitHub API и GameHub).

## Проблемы

### Уведомления не приходят

1. Проверьте, включены ли уведомления в настройках приложения
2. Проверьте системные настройки уведомлений
3. Убедитесь, что приложение не в режиме энергосбережения
4. Проверьте, включены ли уведомления для конкретного репозитория

### Не удается добавить репозиторий

1. Проверьте правильность URL
2. Убедитесь, что репозиторий публичный
3. Проверьте интернет-соединение

### Приложение не проверяет обновления

1. Проверьте интернет-соединение
2. Убедитесь, что не достигнут лимит GitHub API
3. Проверьте настройки фоновой работы приложения
4. Перезапустите приложение

### Не работает темная тема

1. Убедитесь, что "Использовать системную тему" выключена
2. Включите "Темная тема" в настройках
3. Перезапустите приложение

### Приложение вылетает

1. Обновите приложение до последней версии
2. Очистите кэш приложения
3. Переустановите приложение
4. Сообщите об ошибке в Issues с логами
